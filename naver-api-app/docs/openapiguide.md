# 네이버 오픈API 공통 가이드 핵심 요약

본 문서는 네이버 오픈API를 사용하는 클라이언트 애플리케이션 개발 시 필요한 공통 사항을 기술 사양 위주로 요약 정리한 문서입니다.

## 1. 오픈API 종류 및 호출 방식

네이버 오픈API는 인증 방식에 따라 두 가지로 구분됩니다.
- **로그인 방식 오픈 API**: 네이버 아이디로 로그인(OAuth 2.0)을 통해 사용자 인증을 거친 후 호출하는 API (예: 블로그 글쓰기, 카페 글쓰기 등)
- **비로그인 방식 오픈 API**: 로그인 없이 HTTP 요청 헤더에 클라이언트 ID 및 Secret만 전송하여 호출하는 API (예: 검색, 데이터랩 등)

---

## 2. API 호출 인증 헤더 (비로그인 방식)

비로그인 방식 API를 호출할 때는 HTTP 요청 헤더에 아래 두 가지 정보를 반드시 포함해야 합니다.

| HTTP 헤더 키 | 설명 |
| :--- | :--- |
| `X-Naver-Client-Id` | 네이버 개발자 센터에서 애플리케이션 등록 후 발급받은 Client ID |
| `X-Naver-Client-Secret` | 네이버 개발자 센터에서 애플리케이션 등록 후 발급받은 Client Secret |

* API 사용량은 클라이언트 아이디(Client ID)별로 합산되어 일일 제한량이 적용됩니다.

---

## 3. 대표적인 오류 코드 및 해결 방안

| HTTP 상태 코드 | 메시지 / 설명 | 원인 및 대책 |
| :---: | :--- | :--- |
| **400** | Bad Request (잘못된 요청) | 요청 파라미터 값 설정 오류 또는 인코딩 형식(UTF-8) 미준수 |
| **403** | Forbidden (API 권한 없음) | 애플리케이션 등록 시 설정한 **API 권한** 항목에 해당 API가 미포함된 경우. 개발자 센터 콘솔에서 API 설정을 확인해야 합니다. |
| **404** | Not Found (존재하지 않는 API) | 호출 URL의 오타 혹은 잘못된 경로 지정 |
| **429** | Quota Exceeded (호출 한도 초과) | API 일일 호출 제한량을 초과한 경우 |
| **500** | Internal Server Error | 네이버 API 서버 내부 오류 |
