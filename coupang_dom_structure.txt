[쿠팡 카테고리 페이지 DOM 구조 분석 결과]
분석 대상 URL: https://www.coupang.com/np/categories/305798

1. 상품 목록 및 상품 아이템 DOM 구조 요약
- 상품 목록은 일반적으로 ID가 'productList' 이거나 클래스명이 'baby-product-list'인 컨테이너 내부에 위치합니다.
- 각 상품 카드는 <li> 태그로 래핑되어 있습니다.
- <li> 태그 바로 하위에 상품 상세 페이지로 이동하는 <a> 태그가 존재합니다.
  * href 형식: "/vp/products/{productId}?itemId={itemId}&vendorItemId={vendorItemId}&sourceType=CATEGORY&categoryId={categoryId}"
- <a> 태그 내부 구조:
  1) 이미지 영역: <figure> 태그 내부에 상품 이미지 <img> 태그가 위치합니다.
  2) 상세 정보 영역: <div> 태그로 감싸져 있으며 이하의 정보를 포함합니다.
     - 상품명: <div> 태그 내에 일반 텍스트로 상품명이 기록되어 있습니다. (예: "마데카솔 연고, 1개, 8g")
     - 가격 정보: <div> 영역 내에 실제 판매가 <strong>(예: "5,200원"), 로켓배송 여부 이미지 <img>, 단위당 가격 정보 <span>(예: "(1개당 5,200원)")가 위치합니다.
     - 배송 정보: 도착 예정일 텍스트 <span>(예: "내일(목)"), 도착 상태 <span>(예: "도착 보장") 정보가 <div> 형태로 제공됩니다.
     - 별점 및 리뷰 수: 별점 레이아웃을 구성하는 <div>/<span> 요소와 함께 괄호 안에 리뷰 수 <span>(예: "(33940)")가 텍스트로 제공됩니다.

2. __NEXT_DATA__ 스크립트 태그 존재 여부 분석
- 대상 페이지의 URL 패턴('/np/categories/305798')은 쿠팡의 기존 레거시 웹 아키텍처(JSP/Spring 기반)를 사용하고 있음을 나타냅니다.
- 따라서, Next.js 프레임워크의 클라이언트 사이드 하이드레이션을 위한 '__NEXT_DATA__' 스크립트 태그는 본 카테고리 페이지에 존재하지 않습니다.
